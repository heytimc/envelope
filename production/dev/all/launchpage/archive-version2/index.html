Overview

I've just built this great PostgreSQL database. How do I get it to my users?

A full time developer might build a web application. But web applications are complex. Many development environments, web application frameworks and programming languages exist to make web applications possible but none of them make creating a web application easy.

Typically, writing a web application involves learning a programming language for building the server. Which means you need to learn about all the ways a webpage can talk to a server and make sure that you don't introduce security problems. Sometimes there are performance issues or new requirements that could lead to a re-write.

But that's just the server side. Then you've got to write the client. HTML, CSS and what if we need more than just a static web page? There are many Javascript frameworks out there to help because javascript is complex.

Overall, modern web development isn't easy for a database administrator.

Its 2015, we have the technology! I want an easy way to put a website on top of my PostgreSQL database. How easy can it get?

Ideally, the server would be a fixed engine that exposed my database to the local network or even the internet. It would provide a complete API that meant I didn't need to learn server programming at all. I can configure the server to point to my PostgreSQL database, start it up and start writing my web app.

Then, if web browser vendors could supply us with HTML tags to do what we want, like <my-header> or <my-combobox>. Or how about what we really want: <my-table>. That would make our lives a lot easier!

Introducing Envelope Server!

Envelope Server is a complete web app platform that makes building database driven web apps easy with just HTML, basic Javascript and SQL!

Download the Envelope Server and configure it to point at your database cluster. Now you've now got an engine that exposes your database using a fixed API. Security is handled by PostgreSQL so your users log in to Envelope using their PostgreSQL user name and password. From there all requests and security are handled by Envelope Server using encrypted cookies.

But what about the HTML? Until recently, we were stuck with the HTML that browser vendors granted us. Web Components and X-Tags have changed all that. Envelope Server comes with a library of new HTML tags that make building a web app easy. Ever wonder when browser vendors were going to give us a great combo box? Don't wait any longer! Envelope Server comes with all the HTML tags you've been missing, and that makes your HTML a lot easier to read, write and debug. Have a look at an example with attributes removed: 

<gs-page>
    <gs-header>
        <gs-button></gs-button>
        <h3 flex>Detail Page</h3>
        <gs-button></gs-button>
    </gs-header>
    <gs-body>
        <gs-form>
            <template>
               <h3>An example body.</h3>
            </template>
        </gs-form>
    </gs-body>
    <gs-footer>
        <gs-button></gs-button>
        <gs-button></gs-button>
    </gs-footer>
</gs-page>
That's a lot easier to read than your typical web app! Check out the documentation for a complete list of tags and their attributes. There are lots of examples and sample code that you can play with. All our HTML components are mobile aware and make it easy to have your application look good whether its on a phone or on a desktop.
