Description:
It is recommended that you use buttons in place of hyperlinks when the target is an application or download. This way your users will come to expect that buttons 'do things' while hyperlinks lead to a document which will display in the browser. Your users will appreciate consistent application of button and hyperlink usage.

Buttons come with button-appropriate styling such as hover which can be altered using the greyspots css file.

Buttons provide the "onclick" attribute for inline Javascript. Alternately, you can of course use addEventListener('click', function(){}) instead. Additionally, there are href and target attributes like HTML anchor tags.

See below for examples of the many options available for button behavior and styling. Note that the special-use buttons covered in the last section can also use all of the behavior and styling options.

Examples:
Skeleton Example:
A basic gs-button element. Without attributes the button will have the following behavior:
    1) When the button receives mousedown, the appearance chages. It will change back after mouseup or mouseout.
Attribute "onclick":
The onclick attribute allows you to run Javascript as a result of clicking on the button.
Event "click":
Another way of running Javascript from a button is to add a listener to the button. Typically you use a id= attribute to make sure you are listening on the right button.
Attribute "href":
Sometimes you want to open a page like you would a link, but you want the appearance of a button. The target="" and href="" attributes work just like with an <a> tag. This is especially recommended if the target is an application or a download. Note that new windows will either appear as new browser windows or new tabs depending on brower settings.

Behavior and Display Options:
Icons:
Buttons can have icons. These icons can be rotated and moved to different sides inside the button.
Attribute "no-focus":
When the "no-focus" attribute is present the button cannot be tabbed to.
Attribute "key":
In this example we have two buttons with the "key" attribute set. The "key" attribute is for defining a hot-key for a button. Note: you have to click into the example for it to work.
Attribute "no-modifier-key":
In this example we have two buttons with the "key" and "no-modifier-key" attributes set. The "no-modifier-key" attribute means that you can't use CMD or CTRL to activate the hot-key. Note: you have to click into the example for it to work.
Attribute "tabindex":
With this attribute, you can control what order your controls are tabbed to.
Attributes "inline" and "mini":
The inline attribute forces the button to be the smallest size possible with the text. The mini attribute lessens the height of the button and removes the border.
Attribute "disabled":
The disabled attribute greys out the button and prevents click events.
Attribute "emphasis":
The emphasis attribute makes the button stand out by darkening the background color and making the font bolder.
Attribute "remove-" and related:
Sometimes you want the remove the rounded corners on your button. This is usually used with grids.
Special-use Button Examples:
These button elements can't accept "href" and "target" attributes. All other attributes work as expected.
Email Button:
This button will attempt to open a new email in an email client using the HREF="mailto:" attribute of the anchor tag. If the user does not have an email client configured then the behavior is determined by the browser and Operating System. Its possible not all "mailto:" keys (such as BCC) will populate properly for all clients so you should test on your user's system before relying on gs-email.
The gs-email-button is very useful but be aware that in some situations its best to put up a form and send email through the server. For example, on a public website its best to provide a form so that email addresses are not searchable by spammers. On the other hand, if you're writing a private application the user may want all outgoing email to be stored in their sent folder and the only way to do that is to use the "mailto:" API.
Apple Facetime Button:
This button will attempt open an Apple Facetime call. This button uses the "facetime:" link to attempt the call.
Map Button:
This button will open up a map in either Google Maps or Microsoft Bing Maps.
  For Google Maps the button uses this link:
    https://maps.google.com/maps?q=LOCATIONTOSEARCHFOR
  For Microsoft Bing Maps the button uses this link:
    http://www.bing.com/maps/default.aspx?q=LOCATIONTOSEARCHFOR
Phone Button:
This button will make a phone call if a phone line is available. Depending on the browser and operating system: if a phone line is not availible then it may send the link to applications such as Microsoft Skype. This button uses the "tel:" link to attempt the call.
Microsoft Skype Button:
This button will attempt to start a Microsoft Skype call. Microsoft Skype software must be installed for it to work. This button uses the "skype:" link to attempt the call.
Package Tracking Button:
This button will open package tracking pages for some common delivery services.
  For UPS the button uses this link:
    http://www.ups.com/WebTracking/processInputRequest?tracknum=TRACKINGNUMBER
  For USPS the button uses this link:
    https://tools.usps.com/go/TrackConfirmAction?tLabels=TRACKINGNUMBER
  For FEDEX the button uses this link:
    https://www.fedex.com/apps/fedextrack/index.html?tracknumbers=TRACKINGNUMBER
Toggle Button:
This button will toggle between two values when clicked. When getting the value of a gs-toggle like this: TOGGLE.value you will get a Boolean type value.
