Checkbox

 Tag:
<gs-checkbox></gs-checkbox>
Description:
A checkbox, also known as a tickbox is a clickable box that changes between values when clicked. In most cases the values that the checkbox changes between will be "true" and "false" or "-1" and "0". However, there are two other possibilities: "true", "null" and "false" or "-1", "n" and "0".
Examples:
Skeleton Example:
A basic gs-checkbox element. Without attributes the checkbox will have the following behaviors:
    1) The checkbox value will start at "false".
    2) The checkbox value will change between "true" and "false".
Attribute "onchange":
A gs-checkbox element with an "onchange" attribute.
Attribute "value":
Four gs-checkbox elements with the "value" attribute set. The value attribute determines what type the checkbox uses of value: "true" and "false" or "-1" and "0". The javascript in this example is so you can see the value when you click on the checkbox.
Attribute "triplestate":
Four gs-checkbox elements with the "value" and "triplestate" attributes set. The "triplestate" attribute adds the value "null" or "n" depending on the type. The javascript in this example is so you can see the value when you click on the checkbox.
Tabindex Attribute Example: With the tabindex="" attribute, you can control what order your controls are tabbed to.
Attribute "disabled":
A checkbox that has the "disabled" attribute. The "disabled" attribute prevents the checkbox from changing.
Attribute "remove-" and related:
Sometimes you want the remove the rounded corners on a checkbox. This is usually used with grids.
Attributes "inline" and "mini":
The inline attribute forces the checkbox to be the smallest size possible. The mini attribute lessens the height of the button and removes the border.
Attribute "qs":
The name "qs" stands for "querystring". The querystring is the part in the URL after the "?". These checkboxes will get their initial values from the querystring.
