Grid

 Tag:
<gs-grid>
    <gs-block>...</gs-block>
    <gs-block>...</gs-block>
</gs-grid>
Description:
The gs-grid element is used to contain a set of gs-block elements for displaying tabular data. The gs-block elements are similar to cells in Excel and similar to <td> elements in HTML tables.

The gs-grid element is used for setting the width of the gs-block columns. A grid with more blocks than columns will be broken into rows of blocks. When the page gets too thin the grid can make all of it's blocks 're-flow' to display one block per line.

When would you use a grid instead of a table?
    Generally, if you are displaying data, you want <table>. A table element attempts to resize it's columns to show each cell's content. It can also be configured to re-flow using CSS such that labels are repeated for each row. This way each field is identified in the current viewport. You need this when displaying tabular data. For example, ten columns of dates are useless if you don't have a label telling you which date is which.     If you are not displaying tabular data, then you may want a gs-grid. A gs-grid gives you control of your column widths. In the case of layout, you typically want to set your column widths and have them be respected so your layout looks correct. For example, a grid of buttons is a common use for gs-grid. If you set a gs-grid of buttons to re-flow on a narrow screen, they do not need additional labels. They'll look and work properly whether it re-flows or not.
Examples:
Skeleton Example: A basic gs-grid element with three blocks. Without attributes the grid will have the following behaviors:
    1) All three blocks will be equal width.
    2) All three blocks will flow onto their own lines when the viewport is below a certain width.
Attribute "not-responsive": A gs-grid element with the "not-responsive" attribute set and contains three blocks. This grid will have the following behaviors:
    1) All three blocks will be equal width.
    2) No matter what the viewport width is the three blocks will not flow onto their own lines.
Attribute "padded":
The padded attribute on a gs-grid adds a little padding between each column. The width of the grid is not affected and no padding is added before the first column or after the last column. The padded attribute is great for making elements like buttons have a little space between them so they don't look like they are touching.
Attribute "widths":
A gs-grid element with the "widths" attribute will set the widths of its columns. When setting the column widths, the gs-grid will total all the column values and divide each column by the total to get a percentage. In this example, the first column will be set to 20%, the second to 30% and the third to 50%.
Attribute "reflow-at":
In this example we have three gs-grid elements with the "reflow-at" attribute set. The value of the "reflow-at" attribute will determine at what window width the gs-grid element will put it's blocks in one vertical column.
Multiple Rows Example:
A gs-grid element with the "widths" attribute set for three blocks but it contains six blocks. The widths set in the "widths" attribute will be applied to the first three blocks and will be repeated on the second three blocks.
Setting a Block "width" Attribute:
A gs-grid element will use the "widths" attribute to set width attributes only on blocks that do not contain width attributes already. In this example, the fourth and fifth blocks have their "width" attributes pre-set.
Button Example:
A gs-grid element with four blocks. Each contains a button with some or all of its corner rounding removed. In this example, gs-grid causes these buttons to look like a group.
